Cli Command Reference¶

The Main pkm Command¶

pkm - python package management for busy developers

usage: pkm [-h] [-v] [-c CONTEXT] [-g] [-nt] {repos,run,shell,build,vbump,install,uninstall,publish,new,show,clean} ...

Named Arguments¶

-v, --verbose

run with verbose output

Default: False

-c, --context

path to the context to run this command under

-g, --global-context

use the global environment context

Default: False

-nt, --no-tasks

dont run command-attached tasks

Default: False

Contextual Commands¶

Many of pkm’s commands are context dependent, for example, the command:

$ pkm install some_package

installs some_package into some python environment, but which? this depends on the context it runs in:

  • If the above executes inside a virtual environment directory - pkm will install it on that virtual environment.

  • If on the other hand, it executes inside a project directory, pkm will install the dependency inside the virtual environment that belongs to that project (the “attached virtual environment”) and also update the pyproject.toml configuration file.

Finding context¶

The way pkm find the corrent context is very simple, upon execution of context dependent command, pkm looks at the current directory and try to findout if it runs inside a supported context (venv, project, project-group, etc.) if it does - it uses that context, otherwise it goes up a directory and retries. If pkm reaches the root directory it stops and reports that no context could be found for the given command.

Controlling Context¶

if you want pkm to use a specific context (instead of searching for one itself) you can do so using the -c flag. For example,

$ pkm install -c path/to/context some_package

The above command will use the path path/to/context as its context (failing if no supported context found in this path).

Global context¶

pkm considers the environment it was installed in as the global context. If you installed pkm directly on your system installation, the global environment will be that installation. you can ask pkm to use the global context in your commands by using the flag -g, for example:

$ pkm install -g some_package

This command will install the some_package package in the same environment as the one that pkm was installed into.

Package Management Commands¶

This section outlines the pkm sub-commands that are related to package management

pkm install¶

install packages under the current context

usage: pkm install [-h] [-o OPTIONAL] [-f] [-a] [-u] [-m {editable,copy}] [-s {user,system}] [-r REPO] [-R UNNAMED_REPO] ...

Positional Arguments¶

packages

the packages to install (support pep508 dependency syntax)

Named Arguments¶

-o, --optional

optional group to use (only for projects)

-f, --force

forcefully remove and reinstall the given pacakges

Default: False

-a, --app

install package in containerized application mode

Default: False

-u, --update

update the given packages if already installed

Default: False

-m, --mode

Possible choices: editable, copy

choose the installation mode for the requested packages.

Default: “editable”

-s, --site

Possible choices: user, system

applicable for global-context, which site to use - defaults to ‘user’

-r, --repo

bind the given packages to a specific repositry by name, use ‘default’ to remove previous binding

-R, --unnamed-repo

bind the given packages to a new unnamed repositry given its configuration

pkm uninstall¶

remove packages from the current context

usage: pkm uninstall [-h] [-a] [-f] ...

Positional Arguments¶

package_names

the packages to remove

Named Arguments¶

-a, --app

remove containerized packages

Default: False

-f, --force

remove the requested packages even if they are dependant of other packages, will not remove any other packages or update pyproject

Default: False
